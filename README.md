# 🏍️ Bike Store — SENA Project 2025

**Bike Store** es un proyecto completo de **e-commerce moderno**, desarrollado como parte del **Proyecto Formativo SENA (2025)**.  
Combina un **frontend profesional y dinámico** con un **backend estructurado y documentado**, enfocado en ofrecer una experiencia de usuario fluida, atractiva y funcional.

## 📜 Descripción general

**Bike Store** es una plataforma web para la **venta de bicicletas, accesorios y repuestos**, con enfoque en **rendimiento, modularidad y estética moderna**.  
Aplica **buenas prácticas de desarrollo web** tanto en frontend como backend, asegurando escalabilidad, accesibilidad y una arquitectura limpia.

## ⚙️ Tecnologías utilizadas

### 🧩 **Frontend**

- React + TypeScript + Vite
- TailwindCSS / CSS Modules
- React Icons
- Framer Motion (animaciones)
- Context API (gestión de estados globales)

### 🔧 **Backend**

- Node.js + Express
- MySQL
- JWT (autenticación)
- Bcrypt (cifrado de contraseñas)
- Dotenv (variables de entorno)
- CORS y middleware personalizados

## 🚀 Instalación y ejecución del proyecto

### 🔹 1. Requisitos previos

| Herramienta                    | Versión recomendada | Descripción                          |
| ------------------------------ | ------------------- | ------------------------------------ |
| [Node.js](https://nodejs.org/) | ≥ 18.x              | Entorno de ejecución para JavaScript |

| [Git](https://git-scm.com/)    | ≥ 2.x               | Control de versiones                 |

---

### 🔹 2. Clonar el repositorio

```bash
git clone https://github.com/BCoder-X/BCoder-X.BikeStore-SenaProject2025.git
cd BCoder-X-bike-store-senaproject2025
```

---


### 🔹 3. Instalar dependencias del frontend

```bash
cd frontend
npm install
```

---

### 🔹 4. Librerías necesarias (React + TypeScript + Vite)

Si necesitas instalarlas manualmente, ejecuta:

```bash
npm install react-router-dom
npm install lucide-react
npm install react-icons
npm install framer-motion
npm install sweetalert2
...

```

## Expanding the ESLint configuration

If you are developing a production application, we recommend updating the configuration to enable type-aware lint rules:

```js
export default defineConfig([
  globalIgnores(["dist"]),
  {
    files: ["**/*.{ts,tsx}"],
    extends: [
      // Other configs...

      // Remove tseslint.configs.recommended and replace with this
      tseslint.configs.recommendedTypeChecked,
      // Alternatively, use this for stricter rules
      tseslint.configs.strictTypeChecked,
      // Optionally, add this for stylistic rules
      tseslint.configs.stylisticTypeChecked,

      // Other configs...
    ],
    languageOptions: {
      parserOptions: {
        project: ["./tsconfig.node.json", "./tsconfig.app.json"],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
]);
```

You can also install [eslint-plugin-react-x](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-x) and [eslint-plugin-react-dom](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-dom) for React-specific lint rules:

```js
// eslint.config.js
import reactX from "eslint-plugin-react-x";
import reactDom from "eslint-plugin-react-dom";

export default defineConfig([
  globalIgnores(["dist"]),
  {
    files: ["**/*.{ts,tsx}"],
    extends: [
      // Other configs...
      // Enable lint rules for React
      reactX.configs["recommended-typescript"],
      // Enable lint rules for React DOM
      reactDom.configs.recommended,
    ],
    languageOptions: {
      parserOptions: {
        project: ["./tsconfig.node.json", "./tsconfig.app.json"],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
]);
```
